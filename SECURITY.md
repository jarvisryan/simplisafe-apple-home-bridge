# Security policy

## Supported versions

Only the latest tagged release receives security fixes. The current `0.1`
series is alpha software and is not suitable as a sole security control.

## Reporting a vulnerability

Do not open a public issue. Use GitHub's **Report a vulnerability** function in
the repository Security tab. Include the affected version, impact and minimal
reproduction without real credentials, camera images or personal information.

Maintainers should acknowledge a report within seven days and provide an
initial assessment within 14 days. Timelines may change for upstream issues.

Immediately revoke SimpliSafe sessions if a refresh token, OAuth redirect,
signed camera URL or private backup has been exposed.
